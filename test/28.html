What steps will reproduce the problem?
1. Writing "intranet" into omnibar
2. Should automatically resolve into "intranet.mycompany.com" (default dns 
domain)
3. chrome is using a search-engine instead of going to "intranet"

What is the expected result?
Getting the "intranet" page loaded. Not searching for it on the internet.

What happens instead?
Chrome performs a google search for "intranet" instead of going to the 
internal website.

Please provide any additional information below. Attach a screenshot if 
possible.
Comment 1 by huttarl, Sep 02, 2008 
I think this would be considered part of the same issue:
The omnibar is smart but cramming all that functionality into one box only works if 
you really can distinguish between the different kinds of things the user may be 
trying to do.
If you type a URL without the http:// in the omnibar, e.g. code.google.com/chromium, 
and press Enter, you will get a google search for what you typed in, instead of 
actually going to the URL you typed. At least, that happens the first time. After 
you've visited code.google.com/chromium, its presence in your history seems to get 
you the expected result.
Comment 2 by finnur@chromium.org, Sep 02, 2008 
This is by design. 

We default to search because the user is most likely to be searching when typing in a 
single word in the Omnibox. You can force it to always go to the Intranet site by 
adding a slash, such as "intranet/" or wait for Google Chrome to figure out that 
there is also a website reachable with this name. An InfoBar will then pop up saying 
"Did you mean: http://intranet/"? 

If you select yes, the Omnibox will make sure to suggest the Intranet site in the 
future instead of performing a search.
